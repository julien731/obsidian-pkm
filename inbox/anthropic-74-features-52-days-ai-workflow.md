# Anthropic Shipped 74 Features in 52 Days — How We Replicated It

## Anthropic's "Docs to Demos" Workflow

Anthropic's product lead Catherine Wu shared how their team works now:

- **Skip the PRD** — go straight to prototyping
- Build a working prototype with Claude Code in hours
- Ship internally to the whole company
- Watch what people actually do with it (real usage > assumptions)
- Iterate based on that data

**Key stat:** 90% of their code is written by AI. Engineers are shipping PRs with 2,000–3,000 lines fully generated by Claude. The bottleneck has shifted from *engineering capacity* to *deciding what to build*.

---

## Adapted Workflow (from the poster's team)

This team built an internal tool to replicate the approach:

1. **Ideation** — dump notes, emails, call recordings, screenshots into an idea agent that helps PMs curate and prioritize
2. **Planning** — PM + idea agent drafts features using a memory layer (codebase translated into markdown). Collaborative doc: agent drafts → humans comment → agent iterates → repeat
3. **Issues** — issue agent generates tickets from the plan and recommends implementer + priority
4. **Implementation** — humans, agents, or both work issues through: `Backlog → ToDo → In Implementation → Agent Review → In Review`
5. **Review** — PM reviews the build
6. **Merge** — Dev or PM merges

**Result:** Sprint planning overhead drastically reduced. Non-technical product engineers can ship small features end-to-end without depending on a dev.
